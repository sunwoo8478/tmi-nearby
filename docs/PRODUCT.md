# Product Notes

## 문제 정의

동네나 같은 공간에 있는 사람들과 가볍게 말을 섞고 싶지만, 실명 기반 커뮤니티는 부담스럽고 일반 SNS는 너무 넓습니다. TMI Nearby는 아주 짧은 반경 안에서만 보이는 익명 마이크로 피드를 가정한 프로토타입입니다.

## 핵심 사용자 흐름

```text
앱 진입
  → 반경 안의 TMI 카드 확인
  → 좋아요 또는 넘기기
  → 댓글로 짧게 반응
  → 직접 TMI/투표 작성
  → 알림과 내 활동 확인
```

## 주요 기능

| 기능 | 설명 |
| --- | --- |
| Nearby feed | 주변 반경 안의 글만 카드로 표시 |
| Anonymous identity | 매일 바뀌는 익명 닉네임 |
| Quick reaction | 부담 없이 넘기거나 좋아요 |
| Vote card | 가벼운 양자택일 투표 |
| Comment sheet | 글의 맥락을 유지한 하단 시트 댓글 |
| Safety ready | 신고, 차단, 거리 범위 제한을 붙일 수 있는 구조 |

## 다음 개발 후보

- 위치 권한 기반 반경 계산
- 실시간 카드 수신

- 사용자별 익명 닉네임 회전
- 백엔드 API와 데이터베이스 연동
- PWA 설치 지원

## 서비스가 가져야 할 안전장치

익명 서비스는 재미와 동시에 안전장치가 중요합니다. 실제 서비스로 확장한다면 아래 기능을 우선 고려합니다.

| 영역 | 필요 기능 |
| --- | --- |
| 신고 | 게시물, 댓글, 사용자 단위 신고 |
| 숨김 | 내가 보기 싫은 글을 즉시 숨김 |
| 차단 | 특정 익명 세션의 글을 일정 시간 숨김 |
| 거리 제한 | 너무 넓은 범위로 확산되지 않게 반경 제한 |
| 속도 제한 | 짧은 시간 안에 과도한 작성 방지 |
| 민감어 처리 | 욕설, 개인정보, 위치 특정 표현 필터링 |
