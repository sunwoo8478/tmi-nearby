# Product Notes

## 문제 정의

동네나 같은 공간에 있는 사람들과 가볍게 말을 섞고 싶지만, 실명 기반 커뮤니티는 부담스럽고 일반 SNS는 너무 넓습니다. TMI Nearby는 아주 짧은 반경 안에서만 보이는 익명 마이크로 피드를 가정한 프로토타입입니다.

## 핵심 사용자 흐름

```text
앱 진입
  → 반경 안의 TMI 카드 확인
  → 좋아요 또는 넘기기
  → 댓글로 짧게 반응
  → 직접 TMI/투표 작성
  → 알림과 내 활동 확인
```

## 주요 기능

| 기능 | 설명 |
| --- | --- |
| Nearby feed | 실제 위치 기반 거리 계산으로 반경 안의 글만 카드로 표시 |
| Anonymous identity | 매일 바뀌는 익명 닉네임 |
| Quick reaction | 부담 없이 넘기거나 좋아요 |
| Vote card | 양자택일 투표, 선택 후 결과 확인(중복 투표 방지) |
| Comment sheet | 글의 맥락을 유지한 하단 시트에서 댓글 확인과 작성 |
| Safety actions | 신고·숨김·차단 처리와 차단 목록에서 해제(unblock)하는 UI |
| Notifications | 주변 알림 확인과 개별 닫기 |

## 서비스가 가져야 할 안전장치

익명 서비스는 재미와 동시에 안전장치가 중요합니다. 실제 서비스로 확장한다면 아래 기능을 우선 고려합니다.

| 영역 | 필요 기능 | 구현 상태 |
| --- | --- | --- |
| 신고 | 게시물, 댓글, 사용자 단위 신고 | 게시물·댓글·작성자 신고 프론트에 구현 |
| 숨김 | 내가 보기 싫은 글을 즉시 숨김 | 프론트에 구현됨 |
| 차단 | 특정 익명 작성자의 글을 숨김 | 작성자 닉네임 기준으로 프론트에 구현됨(수동 해제 전까지 유지, 시간 제한은 없음) |
| 거리 제한 | 너무 넓은 범위로 확산되지 않게 반경 제한 | 실거리 계산 기준으로 프론트에 구현됨 |
| 속도 제한 | 짧은 시간 안에 과도한 작성 방지 | 프론트에 구현됨 |
| 민감어 처리 | 욕설, 개인정보, 위치 특정 표현 필터링 | 욕설·전화번호·위치 특정 표현(동/호수/아파트 단지명 등) 패턴 프론트에 구현 |

이 표의 "필요 기능"은 서비스가 갖춰야 할 목표치이고, "구현 상태"는 현재 정적 프로토타입(localStorage 기반) 기준입니다. 서버 강제 검증은 [Backend Candidate](./BACKEND.md)의 서버 검증 규칙을 따릅니다.
